## 2024-05-18 - PortfolioLayout Performance Bottleneck\n**Learning:** The `PortfolioLayout` component was repeatedly calculating the sorted list of projects on every render (`const projects = [...(data.projects || [])].sort((a, b) => a.order - b.order);`). This is an O(N log N) operation performed synchronously in the render cycle, which could cause jank, especially as the number of projects grows. \n**Action:** Memoize expensive calculations like sorting large arrays within components using `useMemo` to prevent unnecessary recalculations on re-renders.
